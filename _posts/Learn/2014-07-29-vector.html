1 前言

Vector 是同一种类型的对象的集合，每个对象都有一个对应的整数索引值。

Vector被称为 容器 ，因为它可以包含其他对象。

一个容器中的所有对象必须是同一类型。

使用vector需要包含头文件: 

#include <vector>
using std::vector;

vector本身是一个 类模板 。

2 定义和初始化

几种初始化vector对象的方式: 

vector<T> v1;           // 元素类型是T，默认为空
vector<T> v2(v1);       // 是v1的副本
vector<T> v3(n, i);     // 报告n个值为i的元素
vector<T> v4(n)         // 包含n个元素

3 vector对象的操作

几个比较重要的vector操作: 

v.empty()              // 判断是否为空
v.size()               // 返回元素个数
v.push_back(t)         // 末尾追加值为t的元素，动态增长
v[n]                   // 返回索引为n的元素值
v1 = v2                // 将v1中的元素替换为v2元素的副本
v1 == v2               // 判断相等
!=, <, <=, >, >=       // 保持这些操作符惯有的含义

关于size，类型是: 

vector<T>::size_type   // 必须指明元素类型

同时，要注意 下标索引 只能用于读取vector中元素值，却不能用来定义改变元素值，需要用push_back()来改变； 通过下标赋值时，有时不会引起任何改变.

4 迭代器

迭代器 是一种检查容器内元素并遍历元素的数据类型，比起用下标索引访问容器元素更加通用，基本标准库的每一种容器都有相应的迭代器类型。

4.1 容器的iterator类型

比如vector容器的迭代器类型: 

vector<int>::iterator iter;

4.2 begin和end操作

  • 如果容器非空，则begin函数获取的是第一个元素
  • 如果容器非空，则end函数获取的是一个不存在的元素，即容器最后一个元素的下一个，起到 哨兵 作用，到了end则已经到达了容器的最后一个元素。

4.3 示例

设置容器元素为0 : 

for (vector<int>::iterator iter = ivec.begin(); iter != ivec.end(); ++iter) {
      *iter = 0;
}

or

// 前提是vector已经初始化了空间
for (vector<int>::size_type ix = 0; ix != ivec.size(); ++ix) {
      ivec[ix] = 0;
}

打印元素: 

#include <vector>
#include <iostream>
using namespace std;

int main()
{
    vector<int> ivec = vector<int>(10,4);
    vector<int>::iterator iter;

    for (iter = ivec.begin(); iter != ivec.end(); ++iter){
        cout << *iter << "\n";
    }

    return 0;
}

4.4 const_iterator

鉴于 iterator 可以访问、修改元素的值，因此出现了只可以访问元素的值，不可以修改元素的值的迭代器类型，即 const_iterator ,其它操作和iteraotr一样。

5 顺序容器

vector属于顺序容器，顺序容器包括:

  • vector : 支持快速随机访问
  • list : 支持快速插入/删除
  • deque : 双端队列

5.1 顺序容器的操作

容器定义的类型别名:

  • size_type: 保存容器类型的最大可能容器长度
  • iterator: 迭代器类型
  • const_iterator: 只读迭代器类型
  • reverse_iterator: 逆序寻址元素的迭代器
  • const_reverse_iterator: 只读逆序寻址元素的迭代器
  • different_type: 足够存储两个迭代器差值的有符号整数
  • value_type: 元素类型

5.2 begin和end

操作函数:

  • begin(): 返回迭代器，指向第一个元素
  • end(): 返回迭代器，指向最后一个元素的下一个位置
  • rbegin(): 返回逆序迭代器，指向之后一个元素
  • rend(): 返回逆序迭代器，指向第一个元素的前一个位置

5.3 添加元素

添加元素的操作:

  • push_back(t): 在末尾追加元素，返回void类型
  • push_front(t): 在开头添加元素（只试用于list和deque类型)
  • insert(p,t): 在位置p前面添加元素t
  • insert(p,n,t): 在位置p前面添加n个元素t
  • insert(p,b,e): 在位置p前面添加由迭代器b和e之间的元素

在操作的时候， 应该避免存储end()返回的迭代器 ，因为容器元素的改变后这个迭代器会实现，需要重新加载。

5.4 大小

顺序容器大小的操作:

  • size(): 元素个数，类型是::size_type
  • max_size(): 可容纳的最大元素个数，大于等于size()
  • empty(): 判断是否为0个元素
  • resize(n): 调整大小为n，会删除比n大得元素，否则用新添加的元素初始化新元素(会保留小于n的之前元素)
  • resize(n,t): 调整大小为n，新增的元素值都是t

5.5 访问元素

访问元素的操作:

  • c.back(): 返回最后一个元素的引用
  • c.front(): 返回第一个元素的引用
  • c[n]: 返回下标是n的元素的引用
  • c.at(n): 返回下标为n的元素的引用

5.6 删除元素

删除元素的操作:

  • c.erase(p): 删除迭代器p指向的元素
  • c.erase(b,e): 删除迭代器b和e之间的元素
  • c.clear(): 清空
  • c.pop_back(): 删除最后一个元素
  • c.pop_front(): 删除第一个元素(vector不可用)

5.7 赋值与swap

顺序容器赋值的操作:

  • c1=c2: 删除容器c1的元素，用c2覆盖
  • c1.swap(c2): 互相交换元素，但必须类型相同
  • c.assign(b,e): 将不是指向c的迭代器b和e之间的元素赋值为c
  • c.assign(n,t): 用n个t元素重置容器c

5.8 反转元素、排序、查找

反转: 

#include <algorithm>
reverse(ivec.begin(), ivec.end());

排序: 

#include <algorithm>
sort(ivec.begin(), ivec.end()); // 默认升序

sort(ivec.begin(), ivec.end(), cmp); // 通过cmp来达到降序的结果

查找: 

#include <algorithm>
std::find(vector.begin(), vector.end(), item) != vector.end();

快速初始化: 

static const int arr[] = {16,2,77,29};

vector<int> vec (arr, arr + sizeof(arr) / sizeof(arr[0]) );

or

vector<int> vec2 = { 10, 20, 30 }; // C++11

5.9 其他操作

其他操作有:

  • c.capacity(): 返回容器容量
  • c.~vector<t>(): 销毁容器

6 容器的选择

对于容器类型的选择，根据应用需求来决定:

  • 如果程序需要 随机访问 元素，则选择vector或deque容器
  • 如果程序必须在容器 中间位置 插入或删除元素，则应采用list容器
  • 如果程序不是在中间位置，而是在 首尾插入或删除 元素，则选择deque容器
  • 如果程序只需在读取输入时在容器中间位置插入元素，然后随机访问，则可以先用list容器，然后 排序后 赋值给vector容器

具体还要根据需求具体分析，权衡 访问元素的便利 和 存储代价 。
